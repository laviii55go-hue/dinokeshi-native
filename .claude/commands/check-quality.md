動作確認後のパフォーマンス・セキュリティチェックを実施してください。

## チェック対象

git diff で変更されたファイルすべてを対象にレビューしてください。

## パフォーマンス / キャッシュ チェック項目

以下の観点で問題がないか確認してください：

1. **メモリリーク**: タイマー（setTimeout/setInterval）のクリア漏れ、イベントリスナーの解除漏れ、Ref/Stateの不要な保持
2. **不要な再レンダリング**: React.memoの比較関数の漏れ、Setやオブジェクトの参照変化による不要な再描画
3. **アニメーション負荷**: SharedValue/withTimingの同時大量発火、UIスレッドのブロッキング
4. **非同期処理**: await漏れ、Promise未処理、デバウンス/スロットルの適切さ
5. **キャッシュ**: AsyncStorageの読み書き頻度、Firebase等外部通信のキャッシュTTL
6. **未使用コード**: 未使用の変数・import・関数

## セキュリティ チェック項目

以下の観点で問題がないか確認してください：

1. **機密情報の露出**: APIキー・シークレットのハードコード、デバッグ用コードの残存（DEBUG_START_LEVELなど）
2. **入力バリデーション**: ユーザー入力（プレイヤー名など）のサニタイズ、外部データのパース時のtry-catch
3. **外部通信**: Firebase等への認証なしアクセス、HTTPSの使用
4. **データ永続化**: AsyncStorageに保存するデータの安全性、後方互換のundefinedチェック
5. **インジェクション**: XSS・コマンドインジェクションのリスク（React Nativeでは低リスクだが確認）

## 出力フォーマット

結果を以下の表形式で報告してください：

### パフォーマンス / キャッシュ
| # | 項目 | 判定 | 説明 |

### セキュリティ
| # | 項目 | 判定 | 説明 |

判定は以下の3段階：
- **OK** — 問題なし
- **注意** — 動作には影響しないが改善推奨
- **要修正** — 修正が必要

最後に「要修正」があれば修正提案を、なければ「問題なし」と結論を出してください。
