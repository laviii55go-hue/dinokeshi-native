> **目標ビルド**: 4/1〜2

## 概要

テスターから受領した6件のフィードバックに対応する。
すべてゲーム体験の改善であり、新機能1件（一括消去ボタン）を含む。

**重要**: パフォーマンスを最優先とする。ネイティブアプリのため、演出追加で動作が重くなることは絶対に避けること。

## 対応1: 新恐竜アンロック — ポップアップ廃止・フッター演出に変更

### 現状
- 新恐竜がアンロックされると画像付きモーダルポップアップが表示される（`game.tsx` 937-954行目付近）
- 閉じるボタンを押す必要がありゲームの流れが止まる

### 修正内容
1. **モーダルポップアップを削除**する（`charPopup` 関連のモーダル表示部分）
2. **フッター恐竜エリアの新恐竜アイコンに金色パルスアニメーション**を追加（2秒間）
   - `Animated.loop` で opacity or scale のパルス（1.0→1.3→1.0）を2秒間繰り返し
   - 2秒後に自動停止
3. **自動スクロール**は既存ロジック（`footerListRef.current?.scrollToOffset`）をそのまま維持
4. 恐竜の詳細情報はフッターアイコンタップで確認可能（既存の `dinoInfoPopup` はそのまま残す）

### 対象ファイル
- `app/game.tsx`: モーダル削除、フッター演出追加

## 対応2 & 4: 得点計算の見直し（報酬バランス改善）

### 現状の問題
- タイプ10以降の恐竜は10個以上集めないと消せないが、実際にはほとんど消せていない
- 低レベル恐竜（タイプ0〜6、1〜7個消し）で稼ぐのが最適解になっている
- 高レベル恐竜を消したときの報酬が見合っていない

### 現状の計算式
```
基礎スコア = erasedCount × (cellType + 1)
ボーナス = FIXED_BONUS テーブルから（閾値以上で加算）
合計 = floor((基礎 + ボーナス) × (1 + level × 0.02))
```

### 修正方針
- **基礎計算はそのまま維持**（`erasedCount × (cellType + 1)`）
- **FIXED_BONUSテーブルのタイプ10以降を大幅引き上げ**
- **レアボーナス枠を新設**: 消去必要数が10以上の恐竜を消したとき、固定ボーナスとは別にレア加算

### FIXED_BONUSテーブル修正案

| タイプ | 恐竜名 | 必要消去数 | 現bonus | → 修正後bonus |
|--------|--------|-----------|---------|--------------|
| 0-6 | 基本恐竜 | 1-7 | 10-80 | **据え置き** |
| 7-9 | パキケファロ〜アンキロ | 8-10 | 100-200 | **150-300** |
| 10-12 | マイアサウラ〜マンモス | 11-13 | 250-400 | **500-800** |
| 13-15 | ヒト〜AI | 14-16 | 500-700 | **1000-1500** |
| 16-18 | エイリアン〜ヤマタノオロチ | 17-19 | 800-1200 | **2000-3000** |
| 19-21 | ユニコーン〜麒麟 | 20-22 | 1500-3000 | **4000-6000** |
| 22 | 神 | 23 | 5000 | **10000** |

**注意**: 上記は案であり、実装後に慶さん（チーフ）に確認して最終調整すること。

### 対象ファイル
- `game/logic.ts`: `calcScore()` 内の `FIXED_BONUS` テーブル修正、レアボーナス枠追加

## 対応3: 消去時の落下アニメーション追加

### 現状
- パフォーマンス改善でアニメーションをほぼ無効化しているため、消去後の落下感がない
- ゲームロジック（`applyGravityAndRefill`）の変更は不要

### 修正内容（視覚演出のみ）
1. 消去後に上から埋まるセルに `translateY` アニメーションを追加
   - 開始位置: セル1個分上（`-cellSize`）
   - 終了位置: 元の位置（`0`）
   - 時間: **150〜200ms**
   - イージング: `Easing.out(Easing.quad)`（最初速く→最後減速で「落ちた感」）
2. **位置が変わったセルのみ**アニメーション対象（変わらないセルは動かさない）
3. **`useNativeDriver: true`** で描画をネイティブスレッドに任せる

### パフォーマンス対策
- カクつく場合はアニメーション時間を100msに短縮
- 対象セル数が20以上の場合はアニメーションを間引く（偶数列のみ等）
- 実装後に慶さんの実機（Pixel 7a）で確認し、調整する

### 対象ファイル
- `game/AnimatedCell.tsx`: 落下アニメーション追加

## 対応5: タッチ判定の改善

### 現状
- `react-native-gesture-handler` の `Gesture.Tap()` を使用（`AnimatedCell.tsx`）
- アニメーション中（`animatingRef.current`）はタップを無視
- テスターから「タップしても反応しないことがある」との報告

### 修正内容
1. **`hitSlop` を追加**: タップ判定エリアを上下左右2〜3px拡大
   ```typescript
   const tapGesture = Gesture.Tap()
     .hitSlop({ top: 3, bottom: 3, left: 3, right: 3 })
     .onEnd(() => { runOnJS(onPress)(); });
   ```
2. **アニメーション中のタップキュー**: アニメーション中にタップされた場合、キューに入れてアニメーション完了後に即実行する仕組みを検討
   - ただしパフォーマンスに影響する場合は `hitSlop` のみの対応でOK

### 対象ファイル
- `game/AnimatedCell.tsx`: `hitSlop` 追加、キュー処理検討

## 対応6: 一括消去ボタン「💥ALL」追加（新機能）

### ボタン配置
現在の3ボタン（DEL/MIX/CHG）に4つ目を追加する。

```
現在: [🗑 DEL ×3] [🔀 MIX ×2] [🔄 CHG ×1]
修正: [DEL ×3] [MIX ×2] [CHG ×1] [ALL ×1]
```

- `gap: 6` → `gap: 4` に縮小
- ボタンの絵文字を省略（`🗑 DEL` → `DEL`）
- `fontSize: 15` → `fontSize: 13` に縮小
- 4ボタンとも `flex: 1` で均等分割

### 操作フロー
1. 「ALL」ボタンをタップ → ALLモード ON（ボタンが赤く光る、既存の `itemBtnSelected` スタイル流用）
2. 画面上部に「🔴 ALL: 全消去する恐竜をタップ」のモードテキスト表示（DEL/CHGと同じUI）
3. 盤面の恐竜を1つタップ → そのタイプの恐竜を**盤面から全て消去**
4. 消去アニメーション → 落下 → スコア加算
5. ALLモード解除
6. ボムは対象外（タップしても無効）

### ロジック実装
`game/logic.ts` に `eraseAllOfType()` 関数を新設:
```typescript
export function eraseAllOfType(grid: Cell[][], targetType: number): { newGrid: Cell[][], erasedCount: number } {
  // targetType と同じタイプのセル（bomb以外）を全て消去
  // 消去後に applyGravityAndRefill() を適用
  // 消した個数を返す
}
```

### スコア計算
- 通常の消去と同じ計算（`erasedCount × (cellType + 1)` + FIXED_BONUS）
- 追加で**全消去ボーナス**を加算（消した個数 × 5 程度、要調整）

### 演出
- 発動時: 対象の恐竜が全て同時に光る（0.3秒、既存の `explodingCells` を流用）
- 消滅 → 落下（対応3の落下アニメーションが適用される）

### 獲得条件
- **LV20ごとに1個**獲得
- `checkLevelUp()` に `earnedAll` を追加: `newLevel >= 20 && newLevel % 20 === 0`
- 交換: **DEL×15 → ALL×1**（既存のCHG交換モーダルにALL交換を追加）

### 遊び方ページ更新
アイテム説明テーブルに追加:
```
['LV20〜', '💥ALL', '1種類の恐竜を盤面から全消去']
```

更新履歴に追加:
```
v5.1.0（2026/04/XX）
・一括消去ボタン（ALL）追加
・新恐竜アンロック演出をフッター演出に変更
・高レベル恐竜の報酬バランス見直し
・落下アニメーション追加
・タッチ判定改善
```

### GameState への追加
`game/types.ts` の `GameState` に `allCount: number` を追加。
初期値は `0`。`storage.ts` のシリアライズ/デシリアライズにも対応すること。

### 対象ファイル
- `game/types.ts`: `allCount` 追加
- `game/logic.ts`: `eraseAllOfType()` 新設、`checkLevelUp()` に `earnedAll` 追加、`createInitialState()` に `allCount: 0` 追加
